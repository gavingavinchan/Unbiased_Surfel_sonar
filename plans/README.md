# Plans

This folder holds structured, time-stamped plan documents to track approach and decisions.

## Naming

Use the following convention:

- `plans/PLAN_<NAME>_<YYYY-MM-DD>.md`

Example:

- `plans/PLAN_GUI_POISSON_TUNER_2026-01-17.md`

## Suggested Content

```markdown
# Plan: <Title>

**Date/Time:** YYYY-MM-DD HH:MM:SS CST
**Git Commit:** <current commit hash>

## Goals
- Goal 1
- Goal 2

## Plan
1. Step 1
2. Step 2

## Decisions
For each significant decision, document:
- **What**: The decision made
- **Why**: Reasoning behind the choice
- **Alternatives considered**: Other approaches and why they weren't chosen

## Notes
- Additional context
```

Plans are updated as decisions evolve, then committed with the implementation.
